# PPT 生成功能路线图

## 背景

当前 1.0 版本已经完成“上传 PPT -> 生成讲稿 -> 生成语音 -> 导出页面图片 -> 合成演讲视频”的本地闭环。

下一阶段要把项目从“已有 PPT 生成演讲视频”升级为：

```text
客户资料 / 用户要求 / 参考 PPT
 -> 生成新的 PPT
 -> 生成演讲稿
 -> 生成语音
 -> 合成演讲视频
```

这样项目会更接近真实业务场景：不仅能处理已有 PPT，还能帮助用户批量生产新的客户方案、培训课件、汇报材料和对应讲解视频。

## 总体目标

建立一个“PPT 内容生成 + PPT 文件生成 + 演讲视频生成”的完整工作流。

第一阶段不追求复杂设计和完美版式复刻，先追求：

- 能根据要求生成新的 `.pptx`
- 能参考已有 PPT 的结构和表达风格
- 能下载生成结果
- 能把生成的新 PPT 接入现有讲稿和视频生成流程

## 版本规划

### 1.1：两个入口的最小可用版

1.1 同时提供两个 PPT 生成入口：

1. 从零生成 PPT
2. 参考 PPT 生成新 PPT

这一版的重点是打通流程，而不是追求复杂排版。

#### 模式 A：从零生成 PPT

适用场景：

- 用户只有一个主题和要求
- 没有现成模板
- 想快速生成一份可讲解的 PPT

输入：

- PPT 主题
- 使用场景，例如培训、汇报、销售路演、面试作品
- 目标受众
- 页数
- 风格要求
- 补充说明

输出：

- 新生成的 `.pptx`
- 页面结构清晰，包含标题和正文
- 可继续进入“生成演讲稿预览”
- 可继续进入“一键生成视频”

流程：

```text
用户要求
 -> AI 生成 PPT 大纲
 -> AI 生成每页标题和正文
 -> python-pptx 生成 PPT 文件
 -> 页面展示下载入口
 -> 复用现有演讲稿和视频生成流程
```

#### 模式 B：参考 PPT 生成新 PPT

适用场景：

- 客户上传一份旧 PPT 或模板 PPT
- 希望新 PPT 保持类似结构和表达风格
- 不要求第一版完全复刻复杂版式

输入：

- 参考 PPT
- 新主题或新客户要求
- 目标页数，可默认沿用参考 PPT 页数
- 风格补充说明

输出：

- 新生成的 `.pptx`
- 页面数量和结构尽量参考原 PPT
- 文字内容根据新要求重新生成
- 可继续进入现有演讲视频生成流程

流程：

```text
参考 PPT
 -> 解析标题、正文、页数和结构
 -> 提取参考 PPT 的表达风格
 -> 结合用户新要求生成每页新内容
 -> python-pptx 生成新 PPT
 -> 页面展示下载入口
 -> 复用现有演讲稿和视频生成流程
```

### 1.2：模板复用增强版

1.2 在 1.1 基础上增强“参考 PPT 生成”的效果。

目标：

- 尽量复用参考 PPT 的页面尺寸
- 尽量复用主题色、字体、标题位置
- 尽量保留页面布局
- 在可控范围内替换文本框内容

暂不承诺：

- 复杂动画复刻
- SmartArt 精确复刻
- 复杂图表自动重绘
- 高保真设计级模板生成

### 2.0：批量生成版

2.0 引入批量任务。

典型输入：

- 一个参考模板 PPT
- 一个 Excel 客户列表
- 每行包含客户名、行业、主题、产品、风格、页数等字段

典型输出：

```text
客户A_方案.pptx
客户A_演讲视频.mp4
客户B_方案.pptx
客户B_演讲视频.mp4
客户C_方案.pptx
客户C_演讲视频.mp4
```

流程：

```text
上传 Excel + 模板 PPT
 -> 逐行生成 PPT
 -> 逐个生成讲稿
 -> 逐个生成语音
 -> 逐个合成视频
 -> 打包 zip 下载
```

2.0 需要额外考虑：

- 批量任务进度
- 单个任务失败后不中断全部任务
- 结果目录管理
- zip 打包下载
- 耗时和费用提示
- 日志记录和失败重试

## 建议新增模块

### `app/ppt_outline_generator.py`

职责：

- 根据用户主题和要求生成 PPT 大纲
- 输出结构化大纲，而不是直接输出自然语言

建议输出：

```python
[
    {"slide_index": 1, "title": "标题", "purpose": "这一页要表达什么"},
    {"slide_index": 2, "title": "背景", "purpose": "说明业务背景"},
]
```

### `app/ppt_content_generator.py`

职责：

- 根据大纲生成每页标题、正文、要点
- 为后续生成 PPT 和讲稿提供结构化内容

建议输出：

```python
[
    {
        "slide_index": 1,
        "title": "高速公路工程保险方案",
        "bullets": ["适用场景", "核心保障", "投保流程"],
        "speaker_hint": "这一页用于开场介绍整体方案"
    }
]
```

### `app/ppt_builder.py`

职责：

- 使用 `python-pptx` 从结构化内容生成 `.pptx`
- 第一版使用统一简洁模板
- 后续支持主题色、字体、封面页、目录页等样式配置

### `app/ppt_template_analyzer.py`

职责：

- 解析参考 PPT 的页数、标题结构和正文风格
- 第一版重点提取“结构和文字风格”
- 后续再提取主题色、字体、布局信息

### `app/generated_ppt_storage.py`

职责：

- 管理生成后的 PPT 文件路径
- 支持下载
- 支持把生成的 PPT 继续传给现有视频流程

## 和现有流程的衔接

现有 1.0 流程：

```text
上传 PPT
 -> parse_pptx
 -> generate_scripts
 -> generate_audio_files
 -> export_slides_to_images
 -> compose video
```

加入 PPT 生成后的流程：

```text
生成新 PPT
 -> 保存为 pptx_path
 -> 复用 parse_pptx
 -> 复用 generate_script_preview
 -> 复用一键生成视频
```

关键设计原则：

- PPT 生成只负责产生 `.pptx`
- 后续讲稿、语音和视频仍走现有流程
- 不把“PPT 生成”和“视频生成”混在一个函数里

## 页面设计草案

在 Streamlit 页面中新增一个区域：

```text
生成新 PPT

模式：
  - 从零生成
  - 参考 PPT 生成

输入项：
  - 主题
  - 用途
  - 目标受众
  - 页数
  - 风格要求
  - 补充说明
  - 参考 PPT（仅参考模式需要）

按钮：
  - 生成 PPT

结果：
  - 下载生成的 PPT
  - 使用这个 PPT 生成演讲稿预览
  - 使用这个 PPT 一键生成视频
```

第一版可以先不做复杂多页面 UI，保持在当前 Streamlit 单页中。

## 风险点

### 1. LLM 输出不稳定

风险：

- 模型可能输出非 JSON
- 页数不符合要求
- 每页内容过长或过短

应对：

- 明确要求结构化 JSON
- 增加 JSON 解析失败提示
- 增加字段校验
- 必要时失败后重试一次

### 2. PPT 设计感不足

风险：

- 从零生成的 PPT 可能像普通文档
- 页面排版不够美观

应对：

- 第一版接受“可用优先”
- 提供统一简洁模板
- 后续再优化主题、封面、目录和图文布局

### 3. 参考 PPT 复刻难度高

风险：

- PPT 内部元素复杂
- 文本框、图片、图表、母版、动画都可能不同

应对：

- 1.1 只参考结构和文字风格
- 1.2 再逐步复用布局
- 不承诺复杂动画和高级图表复刻

### 4. 批量生成耗时长

风险：

- 每个 PPT 都要调用 LLM
- 每个视频都要 TTS、导图、ffmpeg

应对：

- 批量功能放到 2.0
- 做任务进度和失败记录
- 允许先只生成 PPT，不立即生成视频

## 1.1 最小实现范围

1.1 只做这些：

- 页面新增“生成新 PPT”区域
- 支持从零生成 PPT
- 支持参考 PPT 生成 PPT
- 生成结果可下载
- 生成结果可作为现有流程输入，继续生成演讲稿和视频
- 增加基础测试

1.1 暂不做这些：

- 批量 Excel
- 高保真模板复刻
- 数字人
- 云端任务队列
- 复杂素材库

## 建议实施顺序

1. 定义生成 PPT 的数据模型
2. 实现从零生成大纲
3. 实现从零生成每页内容
4. 实现 `ppt_builder.py`，把结构化内容写成 `.pptx`
5. 在页面加入“从零生成 PPT”入口
6. 实现参考 PPT 分析
7. 在页面加入“参考 PPT 生成”入口
8. 让生成的新 PPT 进入现有讲稿预览和视频生成流程
9. 补测试和文档

## 面试表达

后续可以这样表达这个升级方向：

> 1.0 版本解决的是“已有 PPT 如何自动生成演讲视频”。下一阶段我把它扩展成“根据客户资料自动生成 PPT，再自动生成演讲视频”的完整内容生产流程。这里的关键不是简单调用大模型，而是把模型输出变成结构化数据，再由程序生成真实 `.pptx` 文件，最后复用已有的语音和视频合成流水线。这样项目从单点工具升级成了一个批量内容生成系统。
